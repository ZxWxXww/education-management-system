### 项目整体规则

项目名称：教培智管系统（EduSmart Manager）
技术栈要求：
- 前端：Vue 3 + Vite + Composition API + <script setup> + JavaScript（禁止使用 TypeScript）
- 后端：Spring Boot 3 + Java 17/21 + MyBatis-Plus + Spring Security + JWT
- 数据库：MySQL 8（开发阶段可使用 H2）
- UI 组件库：Element Plus（优先推荐）或 Naive UI
- 样式：支持深色模式 + 响应式布局（PC为主，兼顾平板）

严格遵守以下规则：

1. 菜单层级必须完全一致
   - 管理员、教师、学生三端菜单必须严格按照以下层级实现（支持折叠子菜单）：

     **管理员端（Admin）**：
     - 仪表盘
     - 用户管理
       - 角色权限管理
     - 课程管理
       - 班级管理
     - 教学资源
       - 作业管理
       - 成绩管理
     - 考勤管理
       - 异常考勤
     - 财务统计
       - 财单管理
     - 数据报表
       - 对比分析
     - 机构设置
       - 日志管理
       - 展示设置

     **教师端（Teacher）**：
     - 教师工作台
     - 我的课程
       - 班级详情
       - 班级管理
     - 教学资源
       - 作业管理
       - 成绩管理
     - 考勤记录
       - 异常考勤
     - 教学数据
       - 成绩分析
     - 个人信息
       - 修改密码

     **学生端（Student）**：
     - 学生学习中心
     - 课程列表
       - 班级信息
     - 我的作业
       - 考试成绩
     - 我的考勤
       - 考勤异常
     - 学习统计
       - 成绩分析
     - 个人信息
       - 修改密码

   - 菜单使用可折叠侧边栏（Sidebar）
   - 支持手风琴式或多展开模式，默认仅展开当前激活的一级菜单
   - 一级菜单为粗体/突出显示，子菜单缩进显示

2. UI 还原要求
   - 必须高度忠实于 Figma 设计稿的布局、颜色、间距、卡片样式、图表、表格、抽屉、弹窗等细节
   - 不允许擅自修改视觉风格、配色、间距和文案
   - 所有页面需支持深色模式切换
   - 表格、表单、统计卡片必须图文并茂，美观且专业

3. 代码规范（前端重点）
   - 前端必须只使用 JavaScript，禁止使用 TypeScript（.vue 文件中 <script setup> 不写 lang="ts"）
   - 使用 Composition API + <script setup> 语法
   - 组件化开发，公共组件需合理抽离（Sidebar、DataTable、StatisticCard、SearchForm 等）
   - API 请求统一封装在 src/api 目录下
   - 使用 Pinia 管理全局状态（用户、权限、主题等）
   - 路由按角色分离（admin、teacher、student）
   - 变量命名使用 camelCase

4. 前后端交互规则
   - 所有接口返回统一格式：{ code: number, message: string, data: any }
   - 分页统一使用 { current, size, total, records }
   - 文件上传仅支持图片和文档（PDF、Word、PPT 等），不支持视频
   - 课时采用“课时包 + 扣减记录”模式

5. 开发流程要求
   - 生成代码时优先按模块输出（先生成侧边栏菜单组件，再生成具体页面）
   - 使用 Mock 数据开发前端，并在代码中清晰注释如何切换到真实后端 API
   - 先生成管理员端，再依次生成教师端和学生端

6. 其他约束
   - 禁止在教学资源和学习资料模块中出现视频相关字段和功能
   - 管理员端不包含任何招生营销相关功能
   - 所有时间字段必须包含 created_at、updated_at
   - 代码必须清晰、可读性高，关键位置添加中文注释

请在后续生成任何代码、组件或页面时，严格遵守以上全部规则。
